2

KIRISH

Bugungi kunda axborot texnologiyalari hayotimizning barcha sohalariga chuqur kirib kelmoqda. Ta'lim sohasida zamonaviy texnologiyalardan foydalanish kundan-kunga ortib bormoqda va maktab boshqaruv jarayonlarini avtomatlashtirish dolzarb masalaga aylangan. O'zbekiston Respublikasi Prezidentining "Raqamli O'zbekiston - 2030" strategiyasiga muvofiq, ta'lim tizimida raqamli texnologiyalarni joriy etish davlat siyosatining muhim yo'nalishlaridan biri hisoblanadi.

Mavzuning dolzarbligi. O'zbekistonda 10 mingdan ortiq umumta'lim maktablari faoliyat yuritib, ularning aksariyatida boshqaruv jarayonlari an'anaviy usulda olib borilmoqda. Raqamli tizimlar o'quv jarayonini real vaqtda kuzatish, o'quvchilar ma'lumotlarini markazlashtirilgan saqlash, ota-onalar bilan aloqani mustahkamlash va hujjat aylanishini kamaytirish imkonini beradi.

Ishning maqsadi - zamonaviy web texnologiyalar (HTML, CSS, JavaScript, Node.js, MongoDB) yordamida maktab boshqaruv tizimini loyihalash va ishlab chiqish. Tizim o'quvchilar va o'qituvchilar ma'lumotlarini boshqarish, dars jadvali tuzish, baholash tizimi, davomat nazorati va hisobotlar shakllantirishni ta'minlashi kerak.

Ishning vazifalari: maktab boshqaruv tizimlarining nazariy asoslarini o'rganish; web texnologiyalarni tahlil qilish va mos texnologiyalarni tanlash; tizim arxitekturasini loyihalash; front-end va back-end qismlarini dasturlash; tizimni sinovdan o'tkazish va joriy etish tavsiyalarini ishlab chiqish.

Tadqiqot ob'ekti - umumta'lim maktablaridagi boshqaruv jarayonlari. Tadqiqot predmeti - web texnologiyalar asosida maktab boshqaruv tizimini ishlab chiqish jarayoni. Tadqiqot usullari: tahliliy, qiyosiy tahlil, modellashtirish va eksperimental usullar qo'llanilgan.

Ishning tuzilishi. Kurs ishi kirish, uchta bob, xulosa va foydalanilgan adabiyotlar ro'yxatidan iborat. Birinchi bobda nazariy asoslar va web texnologiyalar tahlili, ikkinchi bobda tizimni loyihalash va kodlash jarayoni, uchinchi bobda natijalar va tavsiyalar keltirilgan. Ish hajmi 45 betni tashkil etadi.

I BOB. MAKTAB BOSHQARUV TIZIMLARINING NAZARIY ASOSLARI VA WEB DASTURLASH TEXNOLOGIYALARI

1.1. Maktab boshqaruv tizimlarining mohiyati va rivojlanish tarixi

Maktab boshqaruv tizimi (School Management System - SMS) - bu ta'lim muassasalarining kundalik faoliyatini avtomatlashtirish, o'quv jarayonini boshqarish va ma'muriy ishlarni soddalashtirish uchun mo'ljallangan dasturiy ta'minot majmuasidir. Bunday tizimlar o'quvchilar ro'yxatini yuritish, baholarni qayd etish, davomat nazorati, dars jadvali tuzish, moliyaviy hisobotlar shakllantirish va boshqa ko'plab vazifalarni bajarishga qodir.

Maktab boshqaruv tizimlarining paydo bo'lishi XX asrning 80-yillariga borib taqaladi. Dastlabki bosqichda kompyuterlar faqat yirik ta'lim muassasalarida ma'muriy hisob-kitob ishlari uchun qo'llanilgan. 1990-yillarda shaxsiy kompyuterlarning keng tarqalishi bilan maktablarda ham oddiy ma'lumotlar bazalari paydo bo'la boshladi. Biroq bu davrda tizimlar asosan lokal tarmoqlarda ishlagan va faqat ma'muriyat xodimlari foydalangan.

2000-yillardan boshlab internet texnologiyalarining rivojlanishi bilan web-asosidagi maktab boshqaruv tizimlari paydo bo'ldi. Bu tizimlar quyidagi afzalliklarga ega edi:

  • Istalgan joydan internet orqali kirish imkoniyati;
  • Maxsus dasturiy ta'minot o'rnatish zaruratining yo'qligi;
  • Ma'lumotlarni markazlashtirilgan serverda saqlash;
  • Bir vaqtning o'zida ko'p foydalanuvchilar ishlashi mumkinligi;
  • Avtomatik yangilanish va texnik xizmat ko'rsatishning soddaligi.

Bugungi kunda maktab boshqaruv tizimlari bulutli texnologiyalar (cloud computing) asosida ishlaydi va mobil ilovalar orqali ham foydalanish imkonini beradi. Jahon bozorida bir qator mashhur tizimlar mavjud:

1. PowerSchool - AQShda eng ko'p qo'llaniladigan maktab boshqaruv tizimlaridan biri bo'lib, 45 milliondan ortiq o'quvchiga xizmat ko'rsatadi. Tizim o'quv jarayonini to'liq boshqarish, ota-onalar portali, mobil ilova va boshqa ko'plab funksiyalarga ega.

2. Infinite Campus - 8 milliondan ortiq o'quvchiga xizmat ko'rsatuvchi tizim bo'lib, asosan AQSh maktablarida qo'llaniladi. Davomat, baholar, dars jadvali va moliyaviy boshqaruv modullari mavjud.

3. Schoology - ta'limni boshqarish tizimi (Learning Management System) sifatida ham tanilgan bo'lib, o'quv materiallarini tarqatish, vazifalar berish va baholash imkoniyatlarini taqdim etadi.

4

4. EMIS (Education Management Information System) - UNESCO tomonidan qo'llab-quvvatlanadigan ta'lim boshqaruv axborot tizimi bo'lib, asosan rivojlanayotgan mamlakatlarda qo'llaniladi. O'zbekistonda ham EMIS tizimining ayrim elementlari joriy etilgan.

5. Kundalik.uz - O'zbekistonda ishlab chiqilgan elektron kundalik tizimi bo'lib, maktab o'quvchilari, o'qituvchilari va ota-onalar uchun mo'ljallangan. Tizim baholar, davomat va dars jadvali haqida ma'lumot beradi.

Maktab boshqaruv tizimlarining asosiy komponentlari quyidagilardan iborat:

O'quvchilar ma'lumotlar bazasi (Student Information System). Bu modul o'quvchilarning shaxsiy ma'lumotlari, o'quv tarixi, sog'liq holati, oila tarkibi va boshqa muhim ma'lumotlarni saqlaydi. Har bir o'quvchiga individual identifikatsiya raqami beriladi va uning barcha ma'lumotlari yagona profilda jamlangan bo'ladi.

Baholash tizimi (Grading System). Baholash moduli o'qituvchilarga baholarni kiritish, o'quv natijalarini kuzatish va statistik tahlil o'tkazish imkonini beradi. Zamonaviy tizimlarda turli baholash shkalalari (5 balli, 100 balli, A-F tizimi) qo'llab-quvvatlanadi. Shuningdek, formativ va summativ baholash turlari ham ajratilgan bo'ladi.

Davomat nazorati (Attendance Management). Davomat moduli o'quvchilarning darsga qatnashishini qayd etish va kuzatish imkonini beradi. Zamonaviy tizimlarda biometrik identifikatsiya (barmoq izi, yuz tanish), RFID kartalar yoki QR kodlar yordamida avtomatik davomat qayd etish imkoniyatlari mavjud.

Dars jadvali (Timetable Management). Dars jadvali moduli o'qituvchilar, xonalar va sinflar o'rtasidagi muvofiqlikni hisobga olgan holda optimal dars jadvalini tuzish imkonini beradi. Algoritmik yondashuvlar qo'llanilganda, to'qnashuvlar (conflict) avtomatik aniqlanadi va bartaraf etiladi.

Moliyaviy boshqaruv (Financial Management). Moliyaviy modul o'quv to'lovlarini boshqarish, xodimlar ish haqini hisoblash, xarajatlar hisobotini yuritish va byudjet rejalashtirish kabi vazifalarni bajaradi.

5

Aloqa tizimi (Communication Module). Ushbu modul maktab ma'muriyati, o'qituvchilar va ota-onalar o'rtasidagi aloqani ta'minlaydi. SMS xabarlar, elektron pochta xabarnomalar, push-notification va ichki xabar almashish tizimlari orqali barcha manfaatdor tomonlar o'z vaqtida xabardor bo'ladi.

Hisobotlar tizimi (Reporting System). Hisobotlar moduli turli ko'rsatkichlar bo'yicha statistik ma'lumotlarni shakllantirish, grafik va diagrammalar ko'rinishida vizualizatsiya qilish hamda PDF, Excel formatlarida eksport qilish imkonini beradi.

Maktab boshqaruv tizimlarini loyihalashda bir qator muhim tamoyillarga amal qilish zarur:

1. Foydalanish qulayligi (Usability). Tizim interfeysi oddiy va tushunarli bo'lishi kerak, chunki foydalanuvchilar orasida turli yoshdagi va turli texnik saviyaga ega insonlar mavjud. Jakob Nilsenning foydalanish qulayligi tamoyillariga (Nielsen's Usability Heuristics) amal qilish muhim.

2. Xavfsizlik (Security). Ta'lim sohasida shaxsiy ma'lumotlar bilan ishlanganligi sababli, tizim yuqori darajadagi xavfsizlikni ta'minlashi shart. Ma'lumotlarni shifrlash, autentifikatsiya va avtorizatsiya mexanizmlari, HTTPS protokoli va boshqa xavfsizlik choralarini qo'llash zarur.

3. Masshtablanish (Scalability). Tizim foydalanuvchilar soni oshganda ham barqaror ishlashi kerak. Bir nechta o'quvchidan tortib ming lab o'quvchiga qadar xizmat ko'rsata olishi lozim. Buning uchun to'g'ri arxitektura tanlash va optimizatsiya usullarini qo'llash zarur.

4. Ishonchlilk (Reliability). Tizim uzluksiz ishlashi va ma'lumotlar yo'qolmasligi kerak. Zaxira nusxa olish (backup), uzilishlardan tiklash (disaster recovery) va yuqori mavjudlik (high availability) ta'minlanishi lozim.

5. Moslashuvchanlik (Flexibility). Turli maktablar turli talablarga ega bo'lganligi sababli, tizim sozlanishi va moslashtirilishi mumkin bo'lishi kerak. Modular arxitektura bu imkoniyatni ta'minlaydi.

Maktab boshqaruv tizimlarini ishlab chiqishda quyidagi dasturiy ta'minot ishlab chiqish metodologiyalari qo'llaniladi:

Agile metodologiyasi. Agile yondashuvi iterativ va inkremental rivojlantirishga asoslangan. Loyiha kichik bosqichlarga (sprint) bo'linadi va har bir sprint oxirida ishlaydigan mahsulot versiyasi taqdim etiladi. Bu yondashuv maktab boshqaruv tizimlarini ishlab chiqishda juda samarali, chunki talablar o'zgarishi mumkin va tizim bosqichma-bosqich takomillashtiriladi.

Scrum metodologiyasi. Scrum - Agile ning eng mashhur frameworklaridan biri bo'lib, quyidagi rollarni o'z ichiga oladi: Product Owner (mahsulot egasi), Scrum Master (jarayon boshqaruvchisi) va Development Team (ishlab chiqish jamoasi). Sprint planning, daily standup, sprint review va sprint retrospective kabi tadbirlar muntazam o'tkaziladi.

MVC (Model-View-Controller) arxitektura namunasi. Maktab boshqaruv tizimlarini ishlab chiqishda MVC namunasi keng qo'llaniladi. Bu namuna dasturni uchta asosiy komponentga ajratadi:

  • Model - ma'lumotlar va biznes logikasi bilan ishlaydi. O'quvchilar, o'qituvchilar, baholar va boshqa ma'lumotlarni boshqaradi;
  • View - foydalanuvchi interfeysi, ya'ni ma'lumotlarni ko'rsatish va foydalanuvchi kiritmalarini qabul qilish;
  • Controller - Model va View o'rtasidagi aloqani boshqaradi, foydalanuvchi so'rovlarini qayta ishlaydi.

REST (Representational State Transfer) arxitekturasi. Zamonaviy web ilovalar ishlab chiqishda REST API keng qo'llaniladi. REST tamoyillari quyidagilardan iborat:

  • Client-Server arxitekturasi - mijoz va server mustaqil rivojlanadi;
  • Stateless - har bir so'rov mustaqil va oldingi so'rovlarga bog'liq emas;
  • Cacheable - javoblar keshlanishi mumkin;
  • Uniform Interface - yagona interfeys orqali resurslar bilan ishlash;
  • Layered System - qatlamli tizim arxitekturasi.

O'zbekistonda maktab boshqaruv tizimlarining joriy holati va rivojlanish istiqbollarini ko'rib chiqamiz. Mamlakatimizda ta'lim sohasida axborot texnologiyalarini joriy etish bir necha bosqichda amalga oshirilmoqda:

Birinchi bosqich (2000-2010 yillar). Bu davrda maktablarga kompyuterlar yetkazib berildi va informatika fanini o'qitish yo'lga qo'yildi. Biroq boshqaruv jarayonlarini avtomatlashtirish masalasi hali kun tartibiga qo'yilmagan edi. Ba'zi ilg'or maktablarda Microsoft Excel dasturi yordamida oddiy hisobotlar tuzilgan.

Ikkinchi bosqich (2010-2018 yillar). Internet tarmoqlarining kengayishi bilan elektron ta'lim resurslari paydo bo'la boshladi. "ZiyoNET" ta'lim portali ishga tushirildi. Ayrim maktablarda oddiy veb-saytlar yaratildi. Biroq to'liq boshqaruv tizimlari hali mavjud emas edi.

Uchinchi bosqich (2018 - hozirgi vaqt). "Raqamli O'zbekiston" strategiyasi e'lon qilingandan so'ng, ta'lim sohasida keng ko'lamli raqamlashtirish jarayoni boshlandi. Quyidagi muhim qadamlar qo'yildi:

  • Kundalik.uz elektron kundalik tizimi ishga tushirildi;
  • Hemis tizimi oliy ta'lim muassasalari uchun joriy etildi;
  • EMIS tizimi umumta'lim maktablari uchun ishlab chiqildi;
  • O'qituvchilar malakasini oshirish masofaviy platformalari yaratildi;
  • COVID-19 pandemiyasi davrida masofaviy ta'lim platformalari keng joriy etildi.

Shu bilan birga, mavjud tizimlarning bir qator kamchiliklari ham mavjud:

  • Tizimlar ko'pincha bir-biri bilan integratsiya qilinmagan;
  • Foydalanuvchi interfeysi murakkab va noqulay;
  • Mobil qurilmalar uchun moslashtirilmagan;
  • Internet tezligi past bo'lgan hududlarda ishlamaydi;
  • O'zbek tilidagi to'liq qo'llab-quvvatlash mavjud emas;
  • Ma'lumotlar xavfsizligi yetarli darajada ta'minlanmagan.

Ushbu kurs ishida ishlab chiqiladigan maktab boshqaruv tizimi yuqoridagi kamchiliklarni bartaraf etishga qaratilgan. Tizim quyidagi xususiyatlarga ega bo'ladi:

  • Responsive (moslashuvchan) dizayn - barcha qurilmalarda qulay ishlash;
  • O'zbek tilida to'liq interfeys;
  • Zamonaviy va intuitiv foydalanuvchi interfeysi;
  • Yuqori darajadagi ma'lumotlar xavfsizligi;
  • Tez ishlovchi va optimallashtirilgan tizim;
  • Modul tizimi - kerakli funksiyalarni alohida qo'shish imkoniyati.

Maktab boshqaruv tizimlarining samaradorligini baholash mezonlari haqida to'xtalib o'tamiz. Tizim samaradorligini baholashda quyidagi ko'rsatkichlarga e'tibor beriladi:

1. Vaqt tejamkorligi. Tizim joriy etilgandan keyin ma'muriy ishlar uchun sarflanadigan vaqtning qanchalik kamayishi o'lchanadi. Tadqiqotlar shuni ko'rsatadiki, samarali boshqaruv tizimi ma'muriy ishlar vaqtini 40-60% ga kamaytirishi mumkin.

2. Xatolar soni. An'anaviy qo'lda kiritish usulida yo'l qo'yiladigan xatolar soni bilan avtomatlashtirilgan tizimdagi xatolar soni solishtiriladi. Avtomatlashtirish odatda xatolar sonini 80-90% ga kamaytiradi.

3. Foydalanuvchi qoniqishi. O'qituvchilar, ota-onalar va o'quvchilarning tizimdan qoniqish darajasi anketa so'rovlari orqali o'lchanadi. Yaxshi loyihalangan tizim 85% dan yuqori qoniqish ko'rsatkichiga erishishi mumkin.

4. Tizim ishlash tezligi. Web sahifalar yuklanish vaqti, ma'lumotlar bazasiga so'rov javob vaqti va boshqa texnik ko'rsatkichlar o'lchanadi. Optimal ko'rsatkich - sahifa yuklanishi 2 soniyadan oshmasligi kerak.

5. Xavfsizlik darajasi. Tizimning turli xavfsizlik sinovlaridan o'tishi va zaifliklarning yo'qligi tekshiriladi. OWASP Top 10 xavfsizlik tahdidlariga qarshi himoya mavjudligi baholanadi.

Maktab boshqaruv tizimlarida qo'llaniladigan ma'lumotlar modellari haqida batafsil to'xtalamiz. Ma'lumotlar modeli - bu tizimda saqlanadigan ma'lumotlarning strukturasi va ular o'rtasidagi bog'lanishlarni ifodalovchi abstrakt model.

Relyatsion ma'lumotlar modeli. An'anaviy yondashuvda maktab boshqaruv tizimlarida relyatsion ma'lumotlar bazalari (RDBMS) qo'llaniladi. Asosiy jadvallar quyidagicha bo'ladi:

  • Students - o'quvchilar jadvali (id, ism, familiya, tug'ilgan sana, sinf, telefon, manzil);
  • Teachers - o'qituvchilar jadvali (id, ism, familiya, fan, malaka, tajriba);
  • Classes - sinflar jadvali (id, sinf raqami, sinf rahbari, o'quvchilar soni);
  • Subjects - fanlar jadvali (id, fan nomi, sinf, haftalik soat);
  • Grades - baholar jadvali (id, o'quvchi_id, fan_id, baho, sana);
  • Attendance - davomat jadvali (id, o'quvchi_id, sana, holat);
  • Timetable - dars jadvali (id, sinf_id, fan_id, o'qituvchi_id, kun, soat).

NoSQL ma'lumotlar modeli. Zamonaviy yondashuvda MongoDB kabi hujjatga yo'naltirilgan (document-oriented) ma'lumotlar bazalari qo'llanilmoqda. Bu yondashuvda ma'lumotlar JSON formatidagi hujjatlar ko'rinishida saqlanadi. Afzalliklari:

  • Moslashuvchan sxema - yangi maydonlar osongina qo'shiladi;
  • Gorizontal masshtablanish - katta hajmdagi ma'lumotlar bilan ishlash;
  • Yuqori ishlash tezligi - murakkab so'rovlar tez bajariladi;
  • JSON formatida ishlash - web ilovalar uchun qulay.

Ushbu kurs ishida biz gibrid yondashuvni qo'llaymiz - asosiy ma'lumotlar uchun MongoDB (NoSQL) dan foydalanib, murakkab bog'lanishlar uchun esa reference (havola) usulidan foydalanamiz. Bu yondashuv tizimning moslashuvchanligini va ishlash tezligini oshiradi.

Maktab boshqaruv tizimini ishlab chiqishda foydalanuvchi rollariga asoslangan kirish boshqaruvi (Role-Based Access Control - RBAC) muhim ahamiyatga ega. Tizimda quyidagi rollar belgilanadi:

10

Administrator (Super Admin). Tizimning barcha funksiyalariga to'liq kirish huquqiga ega. Foydalanuvchilarni boshqarish, tizim sozlamalarini o'zgartirish va barcha ma'lumotlarga kirish mumkin.

Maktab direktori. Maktab bo'yicha barcha ma'lumotlarni ko'rish, hisobotlarni shakllantirish va muhim qarorlar qabul qilish uchun kerakli ma'lumotlarga kirish huquqi mavjud.

O'qituvchi. O'ziga biriktirilgan sinflar va fanlar bo'yicha baholar qo'yish, davomat belgilash, dars jadvali ko'rish va ota-onalarga xabar yuborish huquqiga ega.

Ota-ona. Farzandining baholarini, davomatini, dars jadvalini ko'rish va o'qituvchilar bilan aloqa qilish imkoniyatiga ega. Boshqa o'quvchilar ma'lumotlarini ko'ra olmaydi.

O'quvchi. O'zining baholarini, dars jadvalini va vazifalarni ko'rish imkoniyatiga ega. Ma'lumotlarni o'zgartira olmaydi.

RBAC tizimini amalga oshirishda JWT (JSON Web Token) texnologiyasidan foydalaniladi. JWT token tarkibida foydalanuvchi identifikatori va rol ma'lumotlari shifrlangan holda saqlanadi. Har bir API so'rovida token tekshiriladi va foydalanuvchi huquqlari aniqlanadi.

Xulosa qilib aytganda, maktab boshqaruv tizimlari ta'lim sohasining muhim tarkibiy qismi bo'lib, ularni zamonaviy web texnologiyalar asosida ishlab chiqish dolzarb vazifa hisoblanadi. Tizim loyihalashda foydalanish qulayligi, xavfsizlik, masshtablanish va ishonchlilik tamoyillariga rioya qilish lozim. Navbatdagi bo'limda biz ushbu tizimni ishlab chiqish uchun zarur bo'lgan web dasturlash texnologiyalarini batafsil ko'rib chiqamiz.

1.2. Zamonaviy web dasturlash texnologiyalari tahlili (HTML, CSS, JavaScript)

Web dasturlash texnologiyalari - bu internet orqali foydalanuvchilarga interaktiv xizmatlar ko'rsatish uchun ishlatiladigan dasturlash tillari, freymvorklar va vositalar to'plamidir. Maktab boshqaruv tizimini ishlab chiqishda front-end (foydalanuvchi tomoni) va back-end (server tomoni) texnologiyalari birgalikda qo'llaniladi.

HTML (HyperText Markup Language). HTML - bu web sahifalarning strukturasini belgilovchi belgilash tili. 1991-yilda Tim Berners-Li tomonidan yaratilgan va hozirgi kunda HTML5 versiyasi keng qo'llanilmoqda. HTML5 quyidagi yangi imkoniyatlarni taqdim etadi:

  • Semantik teglar - header, nav, main, section, article, footer kabi teglar sahifa strukturasini aniqroq ifodalaydi;
  • Form elementlari - email, date, number kabi yangi kiritish turlari validatsiyani soddalashtiradi;
  • Multimedia - video va audio teglari plaginlarsiz media kontentni ko'rsatish imkonini beradi;
  • Canvas va SVG - grafik elementlarni chizish va animatsiyalar yaratish;
  • Web Storage - localStorage va sessionStorage orqali ma'lumotlarni brauzerda saqlash.

Maktab boshqaruv tizimida HTML5 login formalarini yaratish, o'quvchilar ma'lumotlarini kiritish, jadvallar ko'rinishida baholar ko'rsatish, navigatsiya menyusi va hisobotlar sahifalarini tuzish uchun ishlatiladi.

CSS (Cascading Style Sheets). CSS - web sahifalarning ko'rinishini boshqaruvchi stil tili. CSS3 versiyasi zamonaviy web dizayn uchun keng imkoniyatlar taqdim etadi. Asosiy CSS3 xususiyatlari:

  • Flexbox - bir o'lchovli tartibda elementlarni moslashuvchan joylashtirish;
  • CSS Grid - ikki o'lchovli tartibda murakkab sahifa layoutlarini yaratish;
  • Media Queries - turli ekran o'lchamlariga moslashuvchan dizayn;
  • Transitions va Animations - silliq o'tishlar va animatsiyalar;
  • Custom Properties - CSS o'zgaruvchilari orqali stillarni boshqarish.
12

Responsive Design (Moslashuvchan Dizayn). Zamonaviy web ilovalar turli ekran o'lchamlarida (kompyuter, planshet, smartfon) to'g'ri ko'rinishi kerak. Buning uchun CSS Media Queries texnologiyasidan foydalaniladi. Maktab boshqaruv tizimida responsive dizayn ayniqsa muhim, chunki o'qituvchilar va ota-onalar ko'pincha mobil qurilmalardan foydalanadi.

Responsive dizaynning asosiy tamoyillari:

  • Mobile First - avval mobil qurilmalar uchun dizayn, keyin kattaroq ekranlar uchun kengaytirish;
  • Fluid Grids - foizli o'lchamlar yordamida moslashuvchan tarmoq tuzilmasi;
  • Flexible Images - rasmlar konteyner o'lchamiga moslashadi;
  • Breakpoints - ekran o'lchamining muhim nuqtalarida stil o'zgarishlari.

CSS Preprocessorlar (SASS/SCSS). Katta loyihalarda CSS kodini boshqarish murakkablashadi. SASS (Syntactically Awesome Style Sheets) bu muammoni hal qiladi. SASS quyidagi imkoniyatlarni beradi:

  • O'zgaruvchilar - ranglar, shriftlar, o'lchamlarni markazlashtirilgan boshqarish;
  • Nesting - ichma-ich yozish orqali kodni tushunarliroq qilish;
  • Mixins - qayta ishlatiladigan stil bloklarini yaratish;
  • Functions - hisob-kitoblar va mantiqiy amallar bajarish;
  • Partials va Import - stillarni fayllarga bo'lish va modularlashtirish.

Bootstrap Framework. Bootstrap - eng mashhur CSS freymvorklardan biri bo'lib, tayyor komponentlar va responsive grid tizimini taqdim etadi. Maktab boshqaruv tizimida Bootstrap quyidagi maqsadlarda ishlatiladi:

  • 12 ustunli grid tizimi - sahifa layoutini tez yaratish;
  • Tayyor komponentlar - tugmalar, kartalar, jadvallar, modallar;
  • Navigatsiya komponentlari - navbar, breadcrumb, pagination;
  • Form komponentlari - input grouplar, validatsiya stillari;
  • Utility sinflar - margin, padding, ranglar uchun yordamchi sinflar.
13

JavaScript dasturlash tili. JavaScript - bu web sahifalarga interaktivlik qo'shuvchi dasturlash tili. 1995-yilda Brendan Eich tomonidan yaratilgan va bugungi kunda dunyodagi eng mashhur dasturlash tillaridan biri hisoblanadi. ECMAScript 6+ (ES6) standarti JavaScript tiliga ko'plab zamonaviy imkoniyatlarni qo'shdi:

ES6+ yangi xususiyatlari:

  • let va const - blok darajasidagi o'zgaruvchilar e'lon qilish;
  • Arrow Functions - qisqa funksiya yozish sintaksisi;
  • Template Literals - satrlar ichida o'zgaruvchilarni qulay ishlatish;
  • Destructuring - obyekt va massivlardan qiymatlarni ajratib olish;
  • Spread/Rest Operators - massiv va obyektlar bilan ishlash;
  • Promises va Async/Await - asinxron operatsiyalarni boshqarish;
  • Classes - obyektga yo'naltirilgan dasturlash uchun sinf sintaksisi;
  • Modules - import/export orqali kodni modullarga bo'lish.

DOM (Document Object Model) manipulyatsiyasi. JavaScript web sahifadagi elementlarni dinamik ravishda o'zgartirish imkonini beradi. Maktab boshqaruv tizimida DOM manipulyatsiyasi quyidagi holatlarda ishlatiladi:

  • Jadvallarni dinamik to'ldirish va yangilash;
  • Formalarni validatsiya qilish (tekshirish);
  • Modal oynalarni ko'rsatish va yashirish;
  • Navigatsiya menyusini interaktiv qilish;
  • Grafiklar va diagrammalarni chizish;
  • Real-time bildirishnomalar ko'rsatish.

AJAX (Asynchronous JavaScript and XML). AJAX texnologiyasi sahifani qayta yuklamasdan serverdan ma'lumotlar olish imkonini beradi. Fetch API va XMLHttpRequest yordamida serverga so'rov yuboriladi va javob asinxron tarzda qayta ishlanadi. Maktab tizimida AJAX baholarni real vaqtda ko'rsatish, qidiruv natijalarini ko'rsatish va ma'lumotlarni saqlash uchun ishlatiladi.

14

React.js kutubxonasi. React.js - Facebook tomonidan ishlab chiqilgan JavaScript kutubxonasi bo'lib, foydalanuvchi interfeyslari yaratish uchun mo'ljallangan. React komponent asosidagi arxitekturaga ega bo'lib, UI ni qayta ishlatiladigan komponentlarga bo'lish imkonini beradi.

React.js ning asosiy tushunchalari:

  • Components - UI ning mustaqil, qayta ishlatiladigan bo'laklari. Functional va Class komponentlar mavjud;
  • JSX - JavaScript ichida HTML yozish imkonini beruvchi sintaksis kengaytmasi;
  • State - komponentning ichki holati, o'zgarganida UI qayta renderlanadi;
  • Props - ota komponentdan bolaga ma'lumot uzatish;
  • Hooks - useState, useEffect, useContext kabi funksiyalar orqali komponent holati va hayot siklini boshqarish;
  • Virtual DOM - DOM o'zgarishlarini optimallashtirish texnologiyasi.

Maktab boshqaruv tizimida React.js quyidagi afzalliklarni beradi:

  • Komponent asosidagi arxitektura - kodni qayta ishlatish va xizmat ko'rsatishni soddalashtiradi;
  • Virtual DOM - tez renderlash va yuqori samaradorlik;
  • Katta ekosistema - minglab tayyor kutubxonalar va vositalar;
  • React Router - bir sahifali ilova (SPA) navigatsiyasini boshqarish;
  • Redux/Context API - global holatni boshqarish.

Node.js muhiti. Node.js - bu JavaScript kodini serverda ishlatish imkonini beruvchi runtime muhit. 2009-yilda Ryan Dahl tomonidan yaratilgan va Google V8 JavaScript dvigateli asosida ishlaydi. Node.js quyidagi xususiyatlarga ega:

  • Event-driven - hodisalarga asoslangan arxitektura;
  • Non-blocking I/O - blokirovka qilmaydigan kiritish/chiqarish;
  • Single-threaded - bitta oqimda ishlash, lekin asinxron operatsiyalar;
  • NPM - dunyodagi eng katta paket menejeri (2 milliondan ortiq paketlar).
15

Express.js framework. Express.js - Node.js uchun eng mashhur web framework bo'lib, server tomonidagi ilovalarni tez va qulay yaratish imkonini beradi. Express.js minimal va moslashuvchan bo'lib, katta miqdordagi middleware (vositachi dasturlar) va plaginlarni qo'llab-quvvatlaydi.

Express.js ning asosiy xususiyatlari:

  • Routing - URL yo'nalishlarini aniqlash va so'rovlarni boshqarish;
  • Middleware - so'rov va javob orasidagi vositachi funksiyalar;
  • Template Engines - EJS, Pug, Handlebars kabi shablon dvigatellarini qo'llab-quvvatlash;
  • Static Files - statik fayllarni (CSS, JS, rasmlar) xizmat ko'rsatish;
  • Error Handling - xatolarni markazlashtirilgan boshqarish.

Maktab boshqaruv tizimida Express.js quyidagi maqsadlarda ishlatiladi:

  • RESTful API endpointlarini yaratish (GET, POST, PUT, DELETE);
  • Autentifikatsiya va avtorizatsiya middleware larini qo'llash;
  • Ma'lumotlar validatsiyasi (tekshirish);
  • Fayl yuklash (upload) funksiyasini ta'minlash;
  • CORS (Cross-Origin Resource Sharing) sozlamalarini boshqarish.

RESTful API dizayni. REST (Representational State Transfer) - web xizmatlar yaratishning standart arxitektura uslubidir. Maktab boshqaruv tizimi uchun quyidagi API endpointlari loyihalangan:

  • GET /api/students - barcha o'quvchilar ro'yxatini olish;
  • POST /api/students - yangi o'quvchi qo'shish;
  • GET /api/students/:id - ma'lum o'quvchi ma'lumotlarini olish;
  • PUT /api/students/:id - o'quvchi ma'lumotlarini yangilash;
  • DELETE /api/students/:id - o'quvchini o'chirish;
  • GET /api/grades/:studentId - o'quvchi baholarini olish;
  • POST /api/grades - yangi baho qo'yish;
  • GET /api/attendance/:classId/:date - sinf davomatini olish.

Autentifikatsiya va avtorizatsiya texnologiyalari. Web ilovalar xavfsizligini ta'minlashda autentifikatsiya (foydalanuvchini aniqlash) va avtorizatsiya (huquqlarni tekshirish) muhim rol o'ynaydi.

JWT (JSON Web Token). JWT - foydalanuvchi ma'lumotlarini xavfsiz tarzda uzatish uchun ishlatiladigan ochiq standart (RFC 7519). JWT token uch qismdan iborat:

  • Header - token turi va shifrlash algoritmi (masalan, HS256);
  • Payload - foydalanuvchi ma'lumotlari (id, role, exp vaqti);
  • Signature - tokenning haqiqiyligini tasdiqlovchi imzo.

JWT ning afzalliklari: serverda sessiya saqlash zarurati yo'q (stateless), microservices arxitekturasi bilan mos, mobil ilovalar uchun qulay, turli domenlar o'rtasida ishlaydi.

bcrypt parol xeshlash. Foydalanuvchilar parollarini ma'lumotlar bazasida ochiq ko'rinishda saqlash xavfli. bcrypt algoritmi parollarni xeshlash (bir tomonlama shifrlash) uchun ishlatiladi. Salt (tasodifiy qo'shimcha) qo'shilganligi sababli, bir xil parollar ham turli xeshlarni beradi.

Versiyalarni boshqarish - Git. Git - eng mashhur taqsimlangan versiyalarni boshqarish tizimi. Loyiha kodini boshqarish, jamoa bilan ishlash va o'zgarishlar tarixini saqlash uchun ishlatiladi. Asosiy Git buyruqlari:

  • git init - yangi repository yaratish;
  • git add - o'zgarishlarni staging area ga qo'shish;
  • git commit - o'zgarishlarni saqlash;
  • git push - o'zgarishlarni masofaviy serverga yuborish;
  • git pull - masofaviy serverdagi o'zgarishlarni olish;
  • git branch - tarmoqlar bilan ishlash;
  • git merge - tarmoqlarni birlashtirish.

GitHub. GitHub - Git asosida ishlovchi masofaviy kod saqlash platformasi. Ochiq kodli loyihalar, jamoa ishlash va CI/CD (Continuous Integration/Continuous Deployment) jarayonlarini qo'llab-quvvatlaydi.

Ishlab chiqish vositalari va muhitlari. Zamonaviy web dasturlashda quyidagi vositalar keng qo'llaniladi:

Visual Studio Code (VS Code). Microsoft tomonidan ishlab chiqilgan bepul kod muharriri. Keng kengaytmalar tizimi, ichki terminal, Git integratsiyasi va IntelliSense (kod to'ldirish) imkoniyatlariga ega. Maktab boshqaruv tizimini ishlab chiqishda asosiy IDE sifatida ishlatiladi.

Postman. API ni sinovdan o'tkazish va hujjatlashtirish uchun ishlatiladigan vosita. GET, POST, PUT, DELETE so'rovlarni yuborish, javoblarni tekshirish va API to'plamlarini yaratish imkonini beradi.

Chrome DevTools. Google Chrome brauzerining ichki ishlab chiqish vositalari. HTML/CSS ni real vaqtda tahrirlash, JavaScript konsolida kod ishlatish, tarmoq so'rovlarini kuzatish va samaradorlik tahlili o'tkazish imkonini beradi.

NPM (Node Package Manager). Node.js uchun paket menejeri bo'lib, loyiha bog'liqliklarini (dependencies) boshqarish uchun ishlatiladi. package.json fayli loyiha haqidagi barcha ma'lumotlarni va bog'liqliklar ro'yxatini saqlaydi.

Webpack. Zamonaviy JavaScript ilovalar uchun modul bundler (to'plovchi). Turli resurslarni (JS, CSS, rasmlar) birlashtirish, minifikatsiya qilish va optimizatsiya qilish imkonini beradi.

Xulosa qilib aytganda, maktab boshqaruv tizimini ishlab chiqish uchun HTML5, CSS3, JavaScript (ES6+), React.js, Node.js va Express.js texnologiyalari tanlanagan. Bu texnologiyalar zamonaviy, keng qo'llaniladigan va katta hamjamiyat qo'llab-quvvatlaydigan vositalar bo'lib, maktab boshqaruv tizimining barcha funksional talablarini qondirish imkonini beradi. Keyingi bo'limda ma'lumotlar bazasi va server texnologiyalari batafsil ko'rib chiqiladi.
